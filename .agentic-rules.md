# Agentic Rules

## Auto-Commit Hook
Whenever you make changes to the codebase, you **must** run the following check before ending your turn:

1. Check the number of lines changed using `git diff --stat`.
2. If the total number of lines changed (insertions + deletions) is **greater than 100**, you must automatically commit the changes.
3. Use the following command to automate this:
   `powershell -File scripts/auto-commit-large-changes.ps1`

## Commit Message Convention
- For auto-commits, use: `chore: auto-commit large change (>100 lines)`
- Follow with a brief summary of the changes.
